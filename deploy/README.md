# Pharos Sandbox Environments

This directory contains the Podman Compose configurations for the Pharos "Lab-in-a-Box" ecosystem. These environments allow you to run the full stack (Server, Pulse Agent, and Web Console) in a fenced, containerized network.

## 1. Stable Sandbox (`sandbox.yml`)
**Audience:** End-users, Home Labbers, Enterprise Evaluators.
**Behavior:** Pulls pre-built, stable images from GitHub Container Registry (GHCR).
**Usage:**
```bash
# Start the stable sandbox
podman-compose -f deploy/sandbox.yml up -d

# Stop and cleanup
podman-compose -f deploy/sandbox.yml down
```

## 2. Development Sandbox (`sandbox.local.yml`)
**Audience:** Pharos Developers / Contributors.
**Behavior:** Builds container images directly from your **on-host source code**. This ensures that any unsaved or uncommitted changes are reflected in the running environment.
**Use Case:** Use this to verify bug fixes or new features (Human-in-the-Loop) before pushing to GitHub.

**Usage:**
```bash
# Build and start the ecosystem from local source
podman-compose -f deploy/sandbox.local.yml up --build --force-recreate -d

# Verify a fix (example: query the mdb CLI inside the pulse container)
podman exec -it pharos-pulse mdb hostname="pharos-main"

# View logs for a specific service
podman logs -f pharos-web

# Restart a specific service after a code change
podman-compose -f deploy/sandbox.local.yml restart pharos-web

# Stop and cleanup
podman-compose -f deploy/sandbox.local.yml down
```

## Volumes & Persistence
- **pharos-keys:** Persistent volume for the Pharos PKI Root CA and generated admin keys.
- **pharos-pki:** Ephemeral `tmpfs` volume for TLS certificates (regenerated by `pharos-certgen`).
- **pharos-storage:** Ephemeral `tmpfs` volume inside the server for machine/people records (resets on container recreation).
